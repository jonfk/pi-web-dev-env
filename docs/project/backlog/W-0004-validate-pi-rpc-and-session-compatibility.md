# W-0004: Validate Pi RPC and session compatibility

## Status

Backlog

## Depends On

- `W-0001: Spike Pi package family upgrade compatibility`

## Context

Pipane has two relationships to Pi:

- local package imports used for types, session parsing, rendering helpers, and storage/UI shell behavior;
- a child `pi` executable launched in RPC mode, which may be newer than the local packages.

The package upgrade should be validated against real or representative RPC/session behavior, not just TypeScript and build output.

## Goal

Verify that Pipane remains compatible with the selected Pi package family and the supported global `pi` CLI versions used in RPC mode.

## Non-Goals

- Do not implement broad new RPC features unless they are required to preserve existing behavior.
- Do not redesign queue or streaming behavior unless the validation proves the current behavior is incorrect.

## Scope

- Validate JSONL RPC framing assumptions.
- Validate ignored or newly introduced RPC events do not break Pipane.
- Validate session file parsing for sessions produced by newer `pi` versions.
- Validate steering queue behavior against newer RPC event streams.
- Validate extension/tool UI behavior enough to identify unsupported `extension_ui_request` flows.
- Record compatibility expectations for `PI_CLI`, `PATH`, and supported global `pi` versions.

## Acceptance Criteria

- At least one session generated by the selected newer Pi version can be loaded.
- Prompt send and streaming response handling work through RPC.
- Steering while streaming is validated or a specific follow-up bug is filed.
- Unknown and known tool events render without crashing.
- Any newly ignored RPC events are listed with an explicit "safe to ignore" or "needs support" decision.
- A short compatibility note is added under `docs/sessions/` or project docs.
