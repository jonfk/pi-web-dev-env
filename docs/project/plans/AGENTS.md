This directory contains implementation plans for project work.

Each plan should have an id followed by a short stable slug. e.g. `PLAN-001-short-stable-slug.md`.

Do not duplicate content already captured in other artifacts (PRDs, backlog tickets, ADRs, issues, commits, diffs, package documentation). Reference them by path or URL instead.

Once implemented, rejected, or no longer relevant, move the plan to archived and add a status to it.
